# Upgrade

This upgrade guide helps you decide when to move beyond Community.

Upgrade when the work stops being a solo, non-commercial BA workflow.

Common triggers:

- multiple contributors need to contribute or review
- module ownership needs to be split across people
- governance or approval history matters
- you need packaging for handoff or delivery
- you need a publishing integration
- the work needs higher operational discipline across repeated delivery
- the work is client work
- the work becomes commercial or agency-facing

Stay on Community if one operator can run the analysis end to end in markdown.

Move to paid if the workflow needs team structure, governance, packaging, or client-safe delivery.
