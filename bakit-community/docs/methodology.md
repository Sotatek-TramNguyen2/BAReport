# Methodology

BA-kit Community runs one narrow workflow:

`input -> intake -> backbone -> stories -> srs-lite`

Each step has a clear job:

- `input`: capture the raw request, notes, and constraints
- `intake`: normalize the problem, scope, actors, and risks
- `backbone`: define the core requirements and boundaries
- `stories`: turn the backbone into actionable user stories
- `srs-lite`: produce a compact spec for delivery or review

The method is designed for one operator working through a single project lane. It is best for small to medium business analysis work where speed and clarity matter more than governance overhead.

Community intentionally stops before team workflows, approval routing, packaging, and publishing.
